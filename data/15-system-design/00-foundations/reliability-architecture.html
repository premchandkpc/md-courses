🔄 Replication Factor

RF=3 means 3 copies across failure domains. Tolerates N-1 simultaneous failures. Increases write cost 3×.

🔍 Checksums

CRC32 or SHA-256 on every data block. Background scrubbing detects and repairs silent data corruption.

📦 Erasure Coding

Fragment data into M + N parity shards. 4+2 EC tolerates 2 failures with 1.5× storage overhead vs 3× for RF=3.

🗄️ Backup Strategy

Full backup + WAL archiving + point-in-time recovery. Test restores quarterly.
