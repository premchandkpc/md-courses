Network Security Architecture - Defense in Depth

Defense-in-Depth Layers

Strategy: Multiple layers protect against different attacks

DDoS Mitigation Flow

Scenario: 1 Tbps SYN flood attack

Step 1: ISP Edge Detection
Traffic surge detected (1 Tbps vs normal 1 Gbps)
Anomaly threshold exceeded → Activate scrubbing center
Step 2: Scrubbing
- Drop spoofed source IPs
- Rate limit per source
- Filter malformed packets
Result: 99% attack traffic filtered
Step 3: Clean Traffic
Legitimate traffic (~10 Gbps) reaches origin
Service continues normally

WAF Rule Examples

Attack Type Pattern Action

Geo-blocking High-risk country IP Block

Rate Limiting Strategy

Multi-level rate limiting prevents abuse:

  • Per-User: Max 10 requests/sec (prevent bot abuse)
  • Per-IP: Max 100 requests/sec (prevent network floods)
  • Per-Endpoint: Max 1000 requests/sec (prevent targeted attacks)
  • Action: Return 429 Too Many Requests (client retries with backoff)

Key Metrics to Monitor

Security monitoring ensures early detection:

  • ✓ Attack traffic detected (DDoS surge)
  • ✓ Firewall rule hits (unusual patterns)
  • ✓ WAF rule hits (injection attempts)
  • ✓ Response time impact (security shouldn't slow service)
  • ✓ False positive rate (tune to avoid blocking users)
  • ✓ Incident response time (detect to action)
