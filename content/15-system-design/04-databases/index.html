The Real Answer

Most production systems use both. PostgreSQL as source of truth + Redis for caching + Elasticsearch for search + Kafka for event log. The question is not "SQL or NoSQL" but "which tool for which job." Start with PostgreSQL for anything that needs ACID, add specialized stores only when you hit a specific bottleneck.
