Cache Effectiveness

Multi-layer caching dramatically reduces latency:

Latency Sources & Solutions

Source Typical Solution
DNS 10-50ms Caching, GeoDNS
Network propagation Fixed by distance CDN (edge near users)
Router queueing 1-100ms (varies) Rate limiting, compression
App processing 10-1000ms Code optimization, caching
TLS handshake 20-100ms TLS 1.3, session reuse

Bandwidth Saturation Impact

As network fills, latency explodes (queueing theory):

Key Insight: Keep utilization <80% to avoid exponential latency increase

Optimization Priority

Measure → Identify → Optimize (in order):

  1. Compression: Reduce data 50-85% (easiest, high impact)
  2. Connection pooling: Reuse TCP connections (10-100x faster)
  3. CDN: Edge cache near users (4-10x faster)
  4. Protocol upgrade: HTTP/2 multiplexing (2-3x faster)
  5. Capacity: Upgrade links (expensive, last resort)

Key Metrics

✓ Latency Percentiles: P50 (median), P95 (typical user), P99 (worst case)
✓ Packet Loss: Target <0.01% (visible impact >0.1%)
✓ Jitter: Variance in latency (high jitter = queueing)
✓ Bandwidth Utilization: Target ~80% (headroom for spikes)
✓ Per-Hop Latency: Use MTR/traceroute to identify slow segment
