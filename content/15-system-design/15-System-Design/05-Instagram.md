# 05-Instagram

Instagram's system design covers a photo and video sharing platform with feed generation, stories (ephemeral content), direct messaging, content moderation, search, and real-time interactions — serving over 1 billion users on a primarily Python + PostgreSQL stack with heavy caching.

## Key Components
- **Feed Service**: Shows a ranked list of posts from followed users. Generated by pulling from a fan-out-on-write approach — when user A posts, the post is pre-inserted into a feed queue for all of A's followers. This is computationally expensive for users with millions of followers but makes feed reads O(1).
- **Story Service**: Ephemeral content (photos/videos that disappear after 24 hours). Similar fan-out pattern but stored in a list with TTL. Stories are pre-sorted by freshness and the story "ring" at the top of the feed is ranked by machine learning.
- **Direct Messaging**: Two-person and group chats with seen receipts, typing indicators, and media sharing. Uses a separate service with its own infrastructure from the main feed pipeline.
- **Media Upload Pipeline**: Images resized to multiple resolutions, analyzed for NSFW content via hash matching + ML classifiers. Videos transcoded to standard formats with thumbnail generation.
- **Content Moderation**: Deep learning models detect nudity, violence, hate speech, spam. Uses a combination of pre-publish filtering (block obvious violations), post-publish detection (flag for review), and user reporting.

## Key Challenges
- **Feed generation latency**: A user with 1000 followers needs to fan-out 1000 writes per post. For celebrities with 50M+ followers, fan-out is prohibitive — hybrid approach: pre-compute for most followers, pull-from-follow for mega-followers.
- **Caching at scale**: The feed is a cache. Miss rates must be near-zero for a good UX. Cached in Redis clusters (sharded by user ID) with LRU eviction. Warmup on new user login.
- **Storage growth**: 100M+ photos uploaded daily. Need tiered storage (hot: CDN + SSD cache, warm: standard storage, cold: archival). Content-addressable storage (hash of image) for dedup.
- **Stories and ephemerality**: Story views need to be counted but the content is brief-lived. Don't waste permanent storage on 24-hour content.

## Key Design Decisions
- **Fan-out on write** (most users) — write the post to each follower's timeline cache on publish. Makes feed read O(1) at the cost of expensive writes.
- **Fan-out on read** (celebrities) — timeline is generated by pulling recent posts from followed accounts at read time. Avoids writing to millions of queues.
- **Database**: PostgreSQL for core data (users, posts, comments), Cassandra for feed data (high write throughput), Redis for caching and real-time counters, Elasticsearch for search.
- **CDN**: Images served through a custom CDN (or Akamai for smaller markets). Multiple resolution variants generated on upload (150px, 320px, 640px, 1080px).

## Mental Model
Instagram feed generation is like a newspaper delivery service. When an event happens (someone posts), the newspaper prints new editions and delivers them to every subscriber's doorstep (fan-out on write). For celebrities (authors with millions of subscribers), it's more efficient for readers to check the newsstand when they're ready (fan-out on read).
