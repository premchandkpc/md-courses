Scheduler Health Checklist

Scheduling latency — time from process becoming runnable to actually running. Target <10ms for interactive.
Context switch rate — if >50K/s per core, timeslice may be too small.
Run queue depth — sustained >2-3 per core means overloaded system.
Priority distribution — watch for starvation (low-pri never gets CPU).
Per-core utilization — imbalance signals poor load balancing.
