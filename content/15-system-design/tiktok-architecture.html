🎵 TikTok Feed Ranking Architecture

System Overview

🎯 Core Challenge

Generate infinite personalized feed with exploration/exploitation at 2B MAU

Per-scroll ranking. RL-based. Long-term engagement focus.

Scale & Metrics

Traffic

  • 2B+ monthly active users
  • 100+ million creators
  • Scrolls per user: 20/day
  • Videos per scroll: 10 (need new recommendations each time)
  • p99 latency: <500ms

Architecture

User Opens Feed │ ▼ Fast Pool (80%) Exploration Pool (20%) ├─ Personalized ├─ New videos ├─ Known good ├─ New creators └─ High engagement └─ Test content │ │ ├─ Ranking (RL + Bandits) │ ▼ Diversity Filter │ ▼ Feed (10 videos)

Component 1: Contextual Bandits

Thompson Sampling Per Context

  • State: (user, context, video)
  • Action: Show or not show
  • Reward: Watch duration + future engagement
  • Learning: Update theta distribution per context
For each video: ├─ Sample theta ~ Beta(alpha_context, beta_context) ├─ Score = theta (uncertainty bonus for exploration) ├─ Show top-k scored videos └─ Update based on engagement: ├─ If watched: alpha += 1 └─ If skipped: beta += 1

Component 2: Long-Term Value Optimization

Reinforcement Learning

Objective: Maximize lifetime engagement, not just immediate watch-time

  • Short-term: Show viral videos (immediate watch-time)
  • Long-term: Show diverse content (keep user interested)
  • Optimization: 0.7 × short-term + 0.3 × long-term
Reward Function: reward = immediate_watch_time + 0.7 * lambda * predicted_future_sessions Where lambda = decay factor (lower = more immediate focus) Tuned via A/B testing

Component 3: Diversity & Creator Growth

Exploration Allocation

  • 80% known good recommendations
  • 20% exploration (new videos, new creators)
  • Discovery of new interests
  • Creator growth opportunity

Cold Start for New Creators

New creator → allocated to exploration pool → tested with small % of users

If engagement good → gradually increase exposure

Key Metrics

Optimization Targets

  • Watch time per session: How long user stays
  • Session frequency: How often user returns
  • Creator diversity: User sees diverse creators
  • Category diversity: Topic coverage
  • Retention: Long-term usage

A/B Testing

Validation Process

  • New ranking model: Test on 1% users
  • Metrics: Session length, retention, session frequency
  • Duration: 1-2 weeks (need sufficient data)
  • Rollout: If watch-time >= baseline → gradual rollout

🎵 Real-time engagement at 2B scale.

Exploration/exploitation balance. Long-term value optimization.
