HIGH-RISK DOMAINS (Incomplete syntax checks = HIGHER PRIORITY):

1. SQL (12% of all blocks)
   Risk: Multi-dialect differences (PostgreSQL ≠ MySQL ≠ Oracle)
   Solution: sqlglot (multi-dialect parser)
   Coverage: ~85% (requires schema context for full validation)
   Priority: P1 (HIGHEST)

2. BASH (10% of all blocks)
   Risk: Environment-dependent, glob patterns, set -e behavior
   Solution: shellcheck + sh -n fallback
   Coverage: ~90%
   Priority: P1 (HIGHEST)

3. JAVA (8.5% of all blocks)
   Risk: Complex type system, version-specific features (8 vs 11 vs 17)
   Solution: javac with version targeting
   Coverage: ~95%
   Priority: P2 (HIGH)

4. GO (7.7% of all blocks)
   Risk: Package structure, interface satisfaction, goroutine semantics
   Solution: go build with sandbox
   Coverage: ~92%
   Priority: P2 (HIGH)

5. JAVASCRIPT/TYPESCRIPT (17.6% combined)
   Risk: ES6+ features, async/await, type checking (TS)
   Solution: node --check + tsc --noEmit
   Coverage: ~97%
   Priority: P2 (MEDIUM)

6. PYTHON (14.2% of all blocks)
   Risk: Indentation-sensitive, async/await, type hints optional
   Solution: ast.parse() (built-in)
   Coverage: ~98% (type checking optional)
   Priority: P3 (LOW)

================================================================================
                    RISKS & MITIGATIONS
================================================================================

RISK 1: Tool Unavailability
  Impact: Validation skipped for language
  Mitigation: Graceful degradation + warnings in report

RISK 2: False Positives
  Impact: Valid code marked as invalid (especially SQL)
  Mitigation: Manual review of failures, schema-aware validation
  Acceptance: 5-10% false positive rate acceptable

RISK 3: Performance
  Impact: Validation takes too long
  Mitigation: Parallel processing, caching, batch mode
  Target: <5 minutes for full suite

RISK 4: Incomplete Fixes
  Impact: Some failures cannot be automatically fixed
  Mitigation: Document special cases, create improvement backlog
  Acceptance: 90% auto-fixable is sufficient
