Component 1: Feature Collection

Real-Time Features (200 total): ├─ User features (100) │ ├─ Demographics (age, gender, country) │ ├─ CTR history │ ├─ Conversion history │ └─ Engagement metrics │ ├─ Ad features (60) │ ├─ Text embeddings │ ├─ Category │ ├─ Advertiser reputation │ └─ Ad quality score │ └─ Context (40) ├─ Time of day ├─ Device type ├─ Location └─ Seasonality

Component 2: Feature Store (Redis)

Hot Tier (Real-Time)

  • User features: 800 GB (top 2B active users)
  • Ad features: 50 GB (top 100M ads)
  • Cache hit rate: >98%
  • Latency: <5ms p99

Update Pipeline

Source: Kafka impressions stream → Flink aggregation → Redis publish

Frequency: Every 1 minute (sliding window)

TTL: 1 hour (refreshed continuously)

Component 3: CTR Model (DCN)

Deep & Cross Network: User Tower: Ad Tower: Input(100) Input(100) │ │ Dense(512) Dense(512) │ │ Dense(256) Dense(256) │ │ Dense(128) Dense(128) │ │ └─────┬────────────┘ │ Dot Product │ Dense(64) │ Dense(1) + Sigmoid │ CTR [0-1]

Inference

Batch size:10K
Latency:10ms
Throughput:1M rankings/sec
GPUs:300+

Component 4: Online Learning

Feature Updates (Streaming)

  • Per impression: Log to Kafka
  • Aggregate: 1-minute windows
  • Update: User CTR rate, ad CTR rate
  • Write back: Redis cache
  • Latency: <1 minute to reflect in inference

Model Retraining

  • Daily batch: 400M fresh impressions
  • Hourly online: Mini-batch SGD on recent data
  • A/B test: Shadow traffic first (1% users)
  • Rollout: Gradual if metrics improve

Component 5: Fraud Detection

Real-Time Detection

  • Rapid clicks from same IP
  • Click rate > expected CTR
  • Data center IP clicks
  • Unusual device fingerprints
  • Geographic anomalies

Action

High fraud score: Mark click as fraudulent → exclude from billing

False positive rate: 0.5% (acceptable) | False negative: 5%

Bandit Algorithm: Thompson Sampling

For each ad: ├─ Maintain Beta(alpha, beta) distribution │ ├─ alpha = clicks │ └─ beta = skips │ ├─ Ranking: │ └─ Sample theta ~ Beta(alpha, beta) │ └─ Score = theta (higher uncertainty = explore) │ └─ Feedback: ├─ If click: alpha += 1 └─ If skip: beta += 1 Result: Natural exploration/exploitation balance

Production Monitoring

Key Metrics

  • AUC (model quality): Alert if AUC < baseline - 0.01
  • CTR prediction error: Alert if >20% systematic bias
  • Latency: Alert if p99 > 50ms
  • Fraud rate: Alert if spike detected
