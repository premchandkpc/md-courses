🏢 Real-World Optimization Scenarios
E-Commerce Search (100M Products)
Problem: Search with filters takes 2.5s. Solution: Multi-column index + predicate pushdown.
Result: 2.5s → 45ms (55x faster)
Key: Index on (category, price, rating) in query order
Analytics Report (JOIN 5 Tables)
Problem: Report query takes 30s, joins across fact/dimension tables.
Solution: Switch from nested loop to hash join via query hints.
Result: 30s → 3s (10x faster)
Key: Larger table first in hash join
Time-Series Data (1B Events)
Problem: Range query on timestamp slow (full table scan). Solution: Partition by month + index.
Result: 500ms → 15ms (33x faster)
Key: Partition pruning eliminates 97% of data
User Query Aggregation (1M Users)
Problem: GROUP BY slow (full sort). Solution: Use hash aggregate + parallel execution.
Result: 8s → 1s (8x faster)
Key: 8 cores, linear scaling to 1s
Complex WHERE Clause (Conditional Logic)
Problem: Multiple conditions cause index not used. Solution: Rewrite to SARGable form.
Result: 12s → 200ms (60x faster)
Key: Avoid functions in WHERE, use column operators directly
Subquery Optimization (Correlated)
Problem: Correlated subquery runs N times. Solution: Rewrite as JOIN.
Result: 45s → 500ms (90x faster)
Key: Convert EXISTS to JOIN, eliminate repeated execution
