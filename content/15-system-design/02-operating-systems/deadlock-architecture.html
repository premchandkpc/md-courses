Deadlock Architecture
Detection · Prevention · Avoidance · Recovery
Deadlock: threads in a circular wait, each holding a resource another needs. Four Coffman conditions must all hold. Prevention eliminates one condition; detection finds cycles in the wait-for graph via DFS; Banker's algorithm avoids unsafe states by simulating allocation. This page covers all strategies, failure modes, and production scenarios.

Detection vs Prevention

The Real Answer

Most production systems use both. Prevention via lock ordering eliminates deadlock by design. Detection + timeout catches violations from misordered acquisitions. Banker's algorithm is rarely used in practice — too conservative and requires max claims. Start with lock ordering + timeout, add WFG monitoring for production observability.
