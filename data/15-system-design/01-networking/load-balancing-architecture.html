⚖️ Load Balancing Architecture

Distribution of traffic across multiple servers for availability, scalability, and fault tolerance. Layer 4 vs Layer 7, algorithms, and health checks.

Flow: Client → DNS (round-robin) → Load Balancer → Health Check → Backend Pool (N servers) → Response. Failed server = drained from pool.

⚖️ Layer 4 (NLB)

TCP/UDP level. Sees IP + port, not HTTP. Lower latency, fewer features. Connection-level balancing. Preserves client IP. Good for non-HTTP or performance-critical.

📋 Layer 7 (ALB)

HTTP/HTTPS level. Sees URL, headers, cookies, host. Path-based routing (/api → cluster A, /web → B). SSL termination, sticky sessions, WAF.

🔄 Algorithms

Round robin (simple), Least connections (fair), IP hash (sticky), Consistent hash (cache stability), Random, Weighted (different server capacities).

💓 Health Checks

Active (periodic HTTP/TCP checks to /health), Passive (monitor real traffic failures, circuit breaker). Graceful draining: wait for in-flight requests.

Sticky SessionsCookie-based (ALB inserts cookie) or IP hash. Avoid if possible — stateless is better.
Connection Draining30-300s timeout for in-flight requests before terminating. ELB: deregistration_delay.
Cross-ZoneDistribute traffic evenly across AZs, not just within zone. ELB: cross_zone.enabled=true.
Slow StartNew servers start with fewer requests, ramp up over N seconds. Prevents cold-start overload.
