Pattern Recognition — identify the bottleneck from symptoms
System Archetypes
Read-Heavy
Cache everything (Redis, Memcached)
CDN for static assets
Read replicas for DB scaling
Denormalize for query speed
Examples: URL shortener, Twitter timeline, news feed
Write-Heavy
Write-behind cache (async flush)
Kafka / message queue to absorb bursts
LSM-tree DB (Cassandra, RocksDB)
Partition/shard by write key
Examples: IoT sensors, logging, chat messages, metrics
Fan-out Heavy
Fan-out on write (push model) for small graphs
Fan-out on read (pull) for celebrity users
Hybrid: pre-compute followers under 10K
Kafka partitioned by source user
Examples: Twitter, Instagram feed, notification systems
Real-time
WebSocket for bidirectional push
Redis pub/sub for presence
Sticky sessions via consistent hash
Erlang/Go for connection density
Examples: chat, live sports, collaborative editing, gaming

Interview Script Template
1. "Let me clarify the scope. Who's the user? What's the primary action?" 2. "MVP features: [list 3-5]. Out of scope: [list 2-3]." 3. "NFRs I need to clarify: [scale / latency / availability / consistency / durability]." 4. "Given X DAU and Y actions/user, we're looking at ~Z QPS with ~W TB/day storage." 5. "The main constraint is [write throughput / bandwidth / storage / latency]."
✅ Staff+ move: "I'd start monolithic. Here's where I'd split." ✅ Senior move: "If I had to drop one req, it'd be [X] because [Y]."
