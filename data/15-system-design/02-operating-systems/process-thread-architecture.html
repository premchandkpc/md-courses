Process & Thread Management

Process vs Thread

Aspect Process Thread
Memory 10-100MB (isolated) 1-8MB (shared heap)
Creation 100-1000ms 1-5ms
Context Switch 1-10ms (TLB flush) 0.1-1ms (TLB valid)
Isolation Strong (OS protection) Weak (shared memory)

Thread Pool Sizing

Formula: Workers = CPUs × (1 + I/O_time / CPU_time)

  • 8 CPUs, I/O-bound: 8 × (1 + 100/10) = 88 threads
  • 8 CPUs, CPU-bound: 8 × (1 + 0/10) = 8 threads
  • Optimal: Balance utilization without thrashing
  • Too few: Queue backlog, latency spike
  • Too many: Context switch overhead, memory limit

Process Lifecycle

States: Create → Ready → Running → Blocked → Terminated

  • Create: fork/spawn (allocate resources)
  • Ready: waiting for CPU (in queue)
  • Running: executing on core
  • Blocked: waiting for I/O (disk, network, lock)
  • Terminated: exit, cleanup

Context Switch Cost Breakdown

  • Save registers: <1μs (hardware)
  • Load registers: <1μs (hardware)
  • TLB flush: variable (depends on architecture)
  • Cache cold start: 1-10ms (data not in L1)
  • Total overhead: 1-10ms per switch
  • High frequency: >1000 switches/sec = 10% CPU wasted
