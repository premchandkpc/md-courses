🤝 Upgrade Handshake

Client sends GET with Upgrade: websocket, Sec-WebSocket-Key (random base64). Server concatenates with GUID, SHA-1 hashes, returns Sec-WebSocket-Accept.

📦 Frame Format

OPCODE (1=text, 2=binary, 8=close, 9=ping, 10=pong), MASK (client→server must mask), Payload Length (7/16/64 bits), Masking Key, Payload.

🔄 Reconnection

Exponential backoff (1s, 2s, 4s, ... max 30s), jitter. Resume state after reconnect (sequence numbers, subscriptions). Heartbeat via ping/pong.

⚡ Performance

No HTTP overhead per message (vs polling). Single TCP connection. ~2 bytes overhead per frame for small messages.

Protocolws:// (unencrypted), wss:// (TLS encrypted). Runs over TCP.
Frame Overhead2-14 bytes per frame depending on payload size and masking
Max MessageNo spec limit, but 1MB typical. Use streaming for large payloads.
ScalingEach connection consumes ~10-50KB RAM. 10k connections = ~500MB. Use Redis pub/sub for horizontal scale.
