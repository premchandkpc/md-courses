⏸️ Backpressure Architecture

How systems handle overload through rate limiting, load shedding, circuit breakers, and graceful degradation.

Flow: Request → Token Bucket → Circuit Breaker (open/closed/half-open) → Queue → Worker Pool → Downstream.

🚦 Rate Limiting

Token bucket (bursty) or sliding window (smooth). Per-user, per-IP, per-endpoint limits.

⚡ Circuit Breaker

Three states: Closed (normal), Open (fast-fail), Half-Open (test). Resilience4j, Hystrix.

📨 Request Queuing

Bounded queue with overflow policy: drop oldest, newest, random, or return 503.

🛡️ Graceful Degradation

Disable non-critical features, serve stale cached data, reduce search results quality.

Token BucketBurst: accumulate up to max_burst. Steady: refill at rate/s
Circuit BreakerOpen timeout: 30-60s. Half-open test: 1 request
Queue BacklogAlert at 50% capacity. Start shedding at 80%.
Anti-patternsInfinite queue (OOM), no backpressure, retry storm
