BGP Routing Architecture - Border Gateway Protocol

BGP Network Topology

Problem: How route packets between 65K+ independent autonomous systems?

Route Selection Process

BGP picks best route by ranking attributes:

  1. Local Preference: Internal ranking (highest wins)
  2. AS-Path Length: Fewer hops = shorter path
  3. Origin: IGP < EGP < Incomplete
  4. MED: Multi-Exit Discriminator (lowest cost)

Result: Deterministic, policy-driven routing

Route Announcement

AS announces: "I can reach 192.0.2.0/24"

Neighbors learn: AS path = [AS1, AS2, AS3]

Propagation: Takes 3-5 minutes globally

Failover: If direct path fails, use alternate (longer path, higher AS-path)

BGP Convergence Timeline

Scenario: Link between AS2→AS3 fails

t=0s: Link down (detected)
t=5s: AS2 marks route invalid
t=10s: AS2 announces withdrawal
t=25s: Neighbors recalculate
t=30s: New path announced (via backup)
t=180s: Full convergence (up to 3-5 minutes)

Impact: 3-5 minute outage (reason for local failover)
