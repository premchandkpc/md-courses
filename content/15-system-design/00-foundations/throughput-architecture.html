⚖️ Horizontal Scaling

Add more servers behind load balancer. Perfect linear scaling requires stateless apps, shared-nothing database, and distributed caching.

🔌 Connection Pooling

Each DB connection consumes ~10MB RAM. Pool of 20-50 connections per server balances throughput vs resource usage.

📈 Little's Law

L = λ × W: Concurrency = Throughput × Latency. At 10k RPS with 200ms latency, need 2,000 concurrent connections in flight.

⛔ Bottlenecks

CPU-bound: compute/ML inference. I/O-bound: DB queries, network calls. Saturation point: where queue depth grows unbounded.

Horizontal ScalingAdd N servers, throughput × N (up to coordination overhead)
Vertical ScalingBigger instance: 2× CPU/RAM = up to 2× throughput (diminishing returns)
Connection Pool20-50 connections per app instance, monitor for pool exhaustion
ThrottlingRate limit at API gateway (token bucket), backpressure when downstream saturated
