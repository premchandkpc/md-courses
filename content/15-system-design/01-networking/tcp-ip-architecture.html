🔗 TCP/IP Protocol Stack Architecture

The OSI model and TCP/IP stack breakdown — from physical bits to application data, with headers at each layer and how data flows through the stack.

Flow: Application → Transport (TCP/UDP) → Network (IP) → Link (Ethernet/WiFi) → Physical. Each layer adds its own header, the receiver strips them in reverse.

📦 TCP Segment

Source/Dest port (16-bit), Sequence number, ACK number, Flags (SYN/ACK/FIN/RST), Window size, Checksum. 20-60 byte header.

📦 IP Packet

Version (4/6), TTL, Source IP (32-bit v4 / 128-bit v6), Dest IP, Protocol (TCP=6, UDP=17). 20-60 byte header.

📦 Ethernet Frame

Source MAC (48-bit), Dest MAC, EtherType, Payload, FCS (CRC32). 14-22 byte header. Switched at Layer 2.

🔄 Encapsulation

Each layer adds its header around the payload from the layer above. Total overhead: ~66 bytes per packet (TCP over IPv4 over Ethernet).

TCP Overhead20 byte header + 20 byte IP header + 14 byte Ethernet = ~54 bytes/packet
MSSMaximum Segment Size = MTU - IP header - TCP header = 1460 bytes (typical)
MTUMaximum Transmission Unit: Ethernet = 1500 bytes, WiFi = 2304 bytes
Path MTUSmallest MTU along the path. Discovery via ICMP, fragmentation avoided.
