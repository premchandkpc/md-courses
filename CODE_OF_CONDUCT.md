# Code of Conduct

## Our Pledge

We are committed to providing a welcoming, inclusive, and harassment-free environment for everyone, regardless of background, experience level, or identity.

## Our Standards

### Do

- **Be respectful** — Treat others with kindness and professionalism
- **Be constructive** — Provide feedback aimed at improvement, not criticism
- **Be inclusive** — Welcome diverse perspectives and experiences
- **Be collaborative** — Share knowledge and help others learn
- **Focus on quality** — Prioritize accuracy and technical rigor

### Don't

- **Be disruptive** — Harassment, discrimination, or hostile behavior is not tolerated
- **Be dismissive** — Respect others' expertise and avoid gatekeeping
- **Be careless** — Verify facts and examples before contributing
- **Spam** — Don't promote external services or products without relevance

## Standards for Contribution

1. **Technical accuracy** — All code examples should be tested and valid
2. **Real metrics** — Use production numbers, not estimates
3. **Documentation** — Explain the "why" behind technical decisions
4. **Citation** — Reference sources and give credit to original authors
5. **Accessibility** — Make content understandable to learners at all levels

## Scope

This Code of Conduct applies to all spaces managed by this project, including:
- GitHub issues and pull requests
- Discussions and comments
- Contributed content (files, examples, scenarios)

## Enforcement

Instances of unacceptable behavior should be reported by:
1. Opening an issue describing the incident
2. Emailing the project maintainers directly
3. Involving a community moderator if needed

All reports will be reviewed confidentially. The project reserves the right to remove content or contributors that violate this Code of Conduct.
