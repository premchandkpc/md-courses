📺 Netflix Recommendations Architecture

System Overview

🎯 Core Challenge

Cold-start + diversity + long-term satisfaction at 250M users

Batch-driven. Daily recomputation. Focus: retention.

Approach: Offline Batch Pipeline

Nightly Pipeline: 1. Extract Data: ├─ Watch history (past 6 months) ├─ Ratings and feedback └─ User demographics 2. Compute Embeddings: ├─ User embeddings via collab filtering ├─ Content embeddings via collab filtering └─ Update all embeddings 3. Score Phase: ├─ User-content similarity (ANN) ├─ Apply diversity penalties ├─ Re-rank by business rules 4. Storage: ├─ Personalized lists per user ├─ Deploy to serving layer └─ Cache ready for next day

Why Batch?

  • Can use expensive algorithms (collaborative filtering)
  • Can optimize globally (diversity across portfolio)
  • Less infrastructure than real-time
  • Acceptable staleness (24 hours)

Component 1: Collaborative Filtering

User-User Collab

  • Find users similar to target user
  • Aggregate movies they watched
  • Score by (similarity × movie_popularity)

Content-Content Collab

  • Find movies similar to user's watch history
  • Score by (similarity × current_popularity)

Blending

Final score: 0.6 × user-user + 0.4 × content-content

Reduces filter bubble. Better diversity.

Component 2: Cold Start

New User (No Watch History)

  • Onboarding questions: "What genres do you like?"
  • Return: genre-based popular content
  • Gradually personalize as they watch

New Content (No Views)

  • Show in "New & Popular" section
  • Bootstrap engagement
  • Later: surface based on preference match

Component 3: Diversity Optimization

Problem: All recommendations same genre = boredom Solution: Diversity Penalties score = collab_score × (1 - diversity_penalty) diversity_penalty ∝ (# already shown in genre) Example: ├─ Drama score: 0.9 × (1 - 0.5) = 0.45 │ (many dramas already shown) ├─ Comedy score: 0.7 × (1 - 0.1) = 0.63 │ (fewer comedies shown) └─ Comedy ranked higher (despite lower base score!) Result: Spread recommendations across genres

Component 4: Re-ranking & Business Rules

Post-Processing

  • Remove already watched
  • Enforce genre diversity limits
  • Vary content duration
  • Promote new releases
  • Regional content availability

Serving

Precomputed Lists Storage

  • Per user: Top 200 personalized recommendations
  • Storage: Materialized in database
  • Retrieval: O(1) lookup by user_id
  • Cache: Redis for hot users

Daily Refresh

Update cycle: Nightly batch → Deploy new lists → Serve next day

Staleness: Up to 24 hours (acceptable for Netflix)

Key Metrics

Optimization Targets

  • Watch-time: How much user watches
  • Satisfaction: User rating/completion
  • Retention: Long-term subscription
  • Genre diversity: Content variety
  • Discovery: New content consumption

📺 Batch recommendations at scale.

250M users. Daily refresh. Long-term satisfaction focus.
