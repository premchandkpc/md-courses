System Design Principles
Scalability · Availability · Performance · Reliability
System design is building a restaurant empire. Vertical scaling = bigger kitchen. Horizontal = more locations. Load balancer = the host. Caching = pre-made menu items. Consistency = how fast prices update across all locations. CAP = pick two.

Scalability: Vertical vs Horizontal
DimensionVertical (Scale Up)Horizontal (Scale Out)
Key insight: Always prefer horizontal for growth. Start vertical for simplicity, go horizontal when you hit ceiling. Stateless design is prerequisite — move sessions to Redis, use load balancer.
