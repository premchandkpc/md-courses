# System Design Interview

Prepare for system design interviews at top tech companies. Covers 11 real-world systems with architectural deep dives.

## Prerequisites

- [Backend — Junior](backend-junior.md) path or equivalent experience
- Understanding of distributed systems fundamentals

## Path

1. **System Design Principles**
   - [System Design Principles](/15-system-design/01-system-design-principles.md) — 60m
   - Learn: estimation, tradeoffs, consistency models, partitioning

2. **Video Streaming**
   - [Designing Netflix](/15-system-design/02-netflix.md) — 90m
   - Learn: CDN, adaptive bitrate, recommendation, chaos engineering

3. **Messaging**
   - [Designing WhatsApp](/15-system-design/03-whatsapp.md) — 60m
   - Learn: WebSocket, end-to-end encryption, horizontal scaling

4. **Key-Value Store**
   - [Designing DynamoDB](/15-system-design/04-dynamodb.md) — 60m

6. **Video Platform**

7. **Ride Sharing**
   - [Designing Uber](/15-system-design/07-uber.md) — 60m

8. **Social Feed**

    - [Rate Limiting](/15-system-design/08-rate-limiting.md) — 45m

## Interview Tips

- **Framework**: Requirements → Estimation → Data Model → API → High-Level Design → Deep Dive → Tradeoffs
- **Practice**: 3-4 full mocks before the interview
- **Focus areas**: Data partitioning, consistency, fault tolerance, caching

## Outcome

You can whiteboard any major system design question with structured thinking, tradeoff analysis, and production-aware design decisions.
