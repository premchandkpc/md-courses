When to Use DynamoDB

✓ Good Fit

  • Mobile & web apps
  • IoT applications
  • Real-time analytics
  • Session storage
  • Leaderboards
  • User profiles
  • Chat applications
  • Gaming backends

✗ Not Ideal

  • Complex multi-table joins
  • Complex transactions
  • Large aggregations
  • Strong consistency required
  • Complex reporting
  • Heavy analytical queries

Billing Models

On-Demand

  • Pay per request
  • Auto-scales instantly
  • No capacity planning
  • Higher per-unit cost
  • Ideal for variable traffic

Provisioned

  • Fixed capacity units
  • Lower per-unit cost
  • Requires forecasting
  • Auto-scaling available
  • Ideal for predictable load
Model Read Units Write Units Best For
On-Demand $1.25/million reads $6.25/million writes Variable, unpredictable
Provisioned $0.00013/RCU-hour $0.00065/WCU-hour Predictable, high volume

Supported Data Types

Scalar Types

  • String (S)
  • Number (N)
  • Binary (B)
  • Boolean (BOOL)
  • Null

Document Types

  • Map (M) - JSON object
  • List (L) - JSON array

Set Types

  • String Set (SS)
  • Number Set (NS)
  • Binary Set (BS)

Special

  • TTL (Time-to-live)
  • Binary large objects

Read Consistency Models

Eventually Consistent

  • Reads from any replica
  • May return stale data
  • ~50% cheaper
  • Delay: <1 second typical
  • Default behavior

Strongly Consistent

  • Reads from primary
  • Always latest data
  • Double RCU cost
  • For critical operations
  • Must be explicitly chosen
