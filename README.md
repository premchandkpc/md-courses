## The 25 Domains

### 00 — Foundations
CS fundamentals: DS&A, complexity theory, discrete math, programming paradigms. The bedrock of all other domains.

### 01 — AI/ML Engineering
End-to-end AI/ML: math foundations, classical ML, deep learning, transformers, LLMs, agentic AI, production MLOps.

### 02 — Data Engineering
Data infrastructure at scale: batch/stream processing (Spark, Flink), orchestration (Airflow, Dagster), lakehouse, governance.

### 03 — Backend Engineering
Language mastery: Java (JVM, Spring Boot, concurrency), Go (goroutines, scheduler), Python (FastAPI, async), TypeScript (type system).

### 04 — Frontend Engineering
Modern web UIs: React ecosystem, hooks, rendering strategies, state management, SSR (Next.js), performance.

### 05 — Cloud Platforms
Production AWS deep dives (EC2, S3, Lambda, ECS, EKS, RDS, IAM, ElastiCache, CloudWatch) plus GCP/Azure.

### 06 — DevOps & CI/CD
Automated delivery: Terraform, Docker, CI/CD pipelines, GitOps (ArgoCD), configuration management, DevSecOps.

### 07 — Kubernetes
Container orchestration: core K8s, networking (CNI), security (RBAC), storage (CSI), operators, service mesh, performance tuning.

### 08 — Databases
Storage internals: PostgreSQL (MVCC, WAL, vacuum), Redis, MySQL, NoSQL, indexing strategies, query tuning.

### 09 — Distributed Systems
Multi-machine systems: consensus (Raft/Paxos), CAP, replication, distributed transactions, caching, stream processing.

### 10 — Messaging & Streaming
Event-driven backbone: Kafka (internals, partitioning, mirroring), RabbitMQ, SNS/SQS patterns.

### 11 — Networking
Data in motion: TCP/IP, HTTP/1.1-3, QUIC, gRPC, DNS, CDN, load balancing, TLS deep dive.

### 12 — Operating Systems
Linux kernel, CPU scheduling, virtual memory, I/O models (epoll/io_uring), IPC, filesystems, debugging.

### 13 — Security
App/infra security: OWASP Top 10, auth (OAuth 2.0/OIDC), cryptography, zero trust, penetration testing.

### 14 — SRE & Observability
Software engineering for ops: Prometheus, Grafana, OpenTelemetry, SLIs/SLOs, error budgets, incident response.

### 15 — System Design
Scalable architecture: 11 real-world case studies (WhatsApp, Netflix, Twitter, Uber, YouTube, Stripe, Amazon, Discord, Google Search, Zoom, GitHub).

### 16 — Microservices
Loosely coupled services: decomposition, discovery, API gateways, circuit breakers, sagas, distributed tracing.

### 17 — Software Architecture
High-level structure: architecture styles, GoF patterns, DDD, CQRS, event sourcing, architectural decision records.

### 18 — Performance Engineering
Systematic optimization: profiling (CPU/memory/I/O), JVM tuning, latency analysis, flame graphs, benchmarking.

### 19 — Testing
Unit → E2E → chaos → performance testing, contract testing (Pact), property-based testing, flaky test management.

### 20 — Interviews
Structured prep: Java core/concurrency, distributed systems deep dives, system design interview frameworks.

### 21 — Roadmaps
Career navigation: backend progression, distributed systems path, system design interview prep plan.

### 22 — Production Stories
9 real incidents: Kafka outage, PostgreSQL crash, Redis failures, K8s node failure, DNS outage, circuit breaker cascade, distributed deadlock.

### 23 — Projects
Build from scratch: distributed cache, distributed queue, workflow engine. Blueprints with architecture decisions and implementation guidance.

### 24 — Low-Level Design
Detailed OOD: parking lot, elevator system. UML, SOLID, design patterns applied to real problems.

### 25 — Software Engineering
The craft: Git internals, code review, technical writing, collaboration, estimation, project management.

## Learning Paths

### AI/ML Engineer Path
```
00-foundations → 01-ai-ml → 02-data-engineering → 09-distributed-systems
→ 05-cloud → 14-sre-observability → 23-projects → 20-interviews
```
Build ML models and deploy them to production. Start with math fundamentals, then deep learning and LLMs. Add data pipelines for training data, distributed systems for scale, cloud for deployment, and MLOps for reliability.

### Backend Engineer Path
```
00-foundations → 12-operating-systems → 03-backend → 08-databases
→ 11-networking → 16-microservices → 17-software-architecture → 25-software-engineering
→ 18-performance-engineering → 22-production-stories
```
Master server-side development: pick a language (Java, Go, Python, or TypeScript), learn its internals, then layer databases, networking, microservices patterns, and architecture. Complement with real incident experience.

### Data Engineer Path
```
00-foundations → 02-data-engineering → 08-databases → 09-distributed-systems
→ 10-messaging → 05-cloud → 14-sre-observability → 23-projects → 21-roadmaps
```
Build data platforms: start with storage formats and processing engines (Spark, Flink), add messaging for ingestion, cloud for elastic infrastructure, and observability for pipeline health.

### Platform/DevOps Engineer Path
```
12-operating-systems → 06-devops → 07-kubernetes → 05-cloud → 11-networking
→ 13-security → 14-sre-observability → 17-software-architecture → 22-production-stories
```
Design and operate infrastructure: containers, orchestration, CI/CD, cloud native tooling, monitoring, and incident response. Cross-reference with production stories for real failure modes.

### Full-Stack Engineer Path
```
00-foundations → 03-backend → 04-frontend → 08-databases → 11-networking
→ 05-cloud → 16-microservices → 19-testing → 25-software-engineering
```
Build complete applications end-to-end: backend API design, frontend UIs, database modeling, deployment, and testing. Pick one backend lang + React as the default stack.

### System Design Interview Path
```
09-distributed-systems → 15-system-design → 16-microservices → 17-software-architecture
→ 11-networking → 08-databases → 10-messaging → 20-interviews → 21-roadmaps
```
Ace system design interviews: study distributed systems theory, analyze 11 real-world architectures, learn tradeoff articulation. Practice with frameworks from the interview domain.

### Staff+ Engineer Path
```
17-software-architecture → 15-system-design → 18-performance-engineering
→ 13-security → 25-software-engineering → 21-roadmaps → 22-production-stories
→ 14-sre-observability → 24-low-level-design
```
Develop technical leadership: architecture decisions at scale, cross-team influence, incident leadership, performance culture, mentoring, and strategic thinking.

## How to Use This Repository

### Reading Order

Each domain folder has its own README.md with a full table of contents. Start with the domain README, then dive into specific topics based on your learning path above. Documents within a domain are numbered sequentially and can be read in order or referenced independently.

### Search

Use your editor's search (Cmd+Shift+F) or `rg`/`grep` to find specific topics across all 25 domains. Cross-references between domains are embedded as relative links throughout.

### Cross-References

Related domains are linked at the top of each document. For example:
- **Databases** (`08-databases`) ↔ **Distributed Systems** (`09-distributed-systems`) ↔ **Performance Engineering** (`18-performance-engineering`)
- **Kubernetes** (`07-kubernetes`) ↔ **Cloud** (`05-cloud`) ↔ **Networking** (`11-networking`) ↔ **SRE** (`14-sre-observability`)

### Prerequisites

Each document assumes basic familiarity with its topic. For deep dives, the background sections or linked foundational documents provide necessary context.

## Key Features

### Production Focus
Every document emphasizes production realities — not just how things work in theory, but how they behave under load, at scale, and when they fail. Deployment patterns, operational runbooks, and troubleshooting guides are first-class content.

### Failure Analysis
The `22-production-stories` domain contains 9 real-world incidents with full root cause analysis. Every deep-dive document includes failure modes, edge cases, and anti-patterns. Understanding how systems break is as important as knowing how they work.

### Deep Internals
This repository goes beyond surface-level documentation. You'll find PostgreSQL MVCC internals, JVM garbage collection algorithms, Linux kernel scheduler code paths, Kafka log compaction mechanics, Go scheduler source analysis — the layer below the API.

### Interview Preparation
System design interviews, Java coding interviews, and distributed systems deep-dives are covered in dedicated domains. The system design domain includes 11 real-world case studies with tradeoff analysis.

### Visual Learning
The `arch/` directory contains platform-level architecture documents. Each domain README includes structured tables, diagrams (described in text), and progressive disclosure of complexity.

## Contribution & Evolution

This knowledge universe is a living document. Contributions follow these principles:

- **Depth over breadth** — prefer deep internals over shallow overviews
- **Production realism** — include failure modes, edge cases, and operational reality
- **Cross-linking** — every document should reference related domains
- **Progressive disclosure** — start simple, then go deep
- **Evidence-based** — include references, source code links, RFCs

### How to Contribute

1. Identify a gap or improvement in any domain
2. Follow the existing structure and formatting conventions in that domain's README
3. Add cross-references to related domains
4. Include production stories, failure analysis, or code examples where relevant

### Evolution Roadmap

- Interactive simulations for distributed systems algorithms (Raft, gossip, consistent hashing)
- Cheat sheets for quick reference
- Video/visual companions for complex topics
