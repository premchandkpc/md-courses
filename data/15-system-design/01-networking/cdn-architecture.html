CDN Architecture - Content Delivery Networks

CDN Global Distribution

Problem: Serve content fast to 100M users globally (distance = latency)

Cache Hierarchy

Strategy: Multi-layer caching reduces origin load by 95%

1️⃣ Browser Cache → 1ms (if hit)
2️⃣ ISP Cache → 10ms (if hit)
3️⃣ CDN Edge → 50ms (typical hit)
4️⃣ Origin Shield → 100ms (miss on edge)
5️⃣ Origin Server → 200ms (origin fetch)

Result: 95% hit ratio = only 5% reach origin

Latency Impact

Before CDN: User in Tokyo → Origin in SF → 200ms round trip

After CDN: User in Tokyo → Edge in Tokyo → 50ms round trip

Speedup: 4x faster (200ms → 50ms)

Cache Invalidation

Challenge: Update content without invalidating all edges

  • TTL Approach: Wait for expiry (eventual consistency)
  • Versioned URLs: video.mp4 → video-v2.mp4 (no collision)
  • Explicit Purge: API call to CDN (expensive, propagation delay)
  • Stale-While-Revalidate: Serve stale, refresh in background

Key Metrics

Monitor these to ensure CDN health:

  • Cache Hit Ratio: Target >95% (if <80%, check TTL strategy)
  • Edge Latency: P95, P99 (should be <100ms)
  • Origin Latency: Must be <500ms (sign of overload if worse)
  • Error Rate: 4xx, 5xx (track for issues)
