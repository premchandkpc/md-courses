📦 UDP & Datagram Architecture

Connectionless, unreliable, but fast datagram protocol. Foundation for DNS, QUIC, streaming, gaming, and real-time communication.

Flow: Application → UDP (no handshake, no ordering, no retransmission) → IP → Destination. Fire-and-forget with optional application-level reliability.

📦 UDP Header

Only 8 bytes: Source Port (16-bit), Dest Port (16-bit), Length (16-bit), Checksum (16-bit). No sequence numbers, no ACKs, no windowing.

🚀 QUIC (HTTP/3)

Barely-UDP — adds TCP-like reliability at application layer. 0-RTT handshake, connection migration, stream multiplexing without HOL blocking.

📞 WebRTC (SRTP/SCTP)

Real-time audio/video over UDP. SCTP for data channels (reliable ordered). SRTP for media (tolerates packet loss, values low latency).

🌐 DNS over UDP

Single UDP request/response (max 512 bytes, EDNS0 extends to 4096). Falls back to TCP for large responses (zone transfers).

Header Size8 bytes (vs 20-60 for TCP). 0 overhead for connection state.
Max Datagram65535 bytes including header. Path MTU typically 1500, so fragmentation common.
Use CasesDNS, DHCP, NTP, VoIP, video streaming, gaming, QUIC, WebRTC, DTLS
Application ResponsibilitySequencing, retransmission, congestion control, flow control — all at app layer
