Storage Engines

✓ InnoDB (Recommended)

  • ACID transactions
  • Foreign keys
  • Crash recovery
  • Row-level locking
  • Better concurrency
  • Default since 5.5

⚠️ MyISAM (Legacy)

  • Fast for reads
  • No transactions
  • Table-level lock
  • Lower overhead
  • Use for legacy only
  • Not recommended

When to Use MySQL

✓ Good For

  • Web applications
  • E-commerce platforms
  • CMS systems
  • Financial apps
  • User authentication
  • Transactional data

✗ Not For

  • Extreme distribution

Table Constraints

Constraint Purpose Example
PRIMARY KEY Unique row identifier userId INT PRIMARY KEY
UNIQUE No duplicate values email VARCHAR(255) UNIQUE
NOT NULL Required field firstName VARCHAR(100) NOT NULL
FOREIGN KEY Relationship to another table FOREIGN KEY (userId) REFERENCES users
DEFAULT Default value status VARCHAR(20) DEFAULT 'active'
CHECK Validate condition age INT CHECK (age >= 18)

Indexing for Performance

Primary Key

  • Fastest lookup
  • One per table
  • Unique required
  • ID-based queries

Unique Index

  • Email, username
  • No duplicates
  • Very fast
  • Enforce uniqueness

Regular Index

  • WHERE clauses
  • JOIN conditions
  • Sort operations
  • Multiple per table

Composite Index

  • Multiple columns
  • Efficient ranges
  • Query-specific
  • Order matters

ACID Transactions

Atomicity

All operations succeed or all rollback. No partial updates.

Consistency

Database valid before and after transaction. Constraints enforced.

Isolation

Concurrent transactions don't interfere. Independent execution.

Durability

Committed data survives failures. Persistent storage.
